# Чек-лист ручного QA — Филка

Документ описывает сценарии end-to-end проверки приложения после редизайна под дизайн-систему «Филка».

## 0. Предусловия

- Бекенд на `http://localhost:13500/api` запущен (`go run ...`).
- Фронтенд: `npm run dev` (порт 3000).
- В .env: `NEXT_PUBLIC_API_URL=http://localhost:13500/api`, `NEXT_PUBLIC_WS_URL=ws://localhost:13500/api/ws`.
- Бекенд в режиме `Env != production` для прохождения SMS-верификации любым 6-значным кодом.

## 1. Регистрация заказчика

1. Открыть `/`.
2. На лендинге — нажать «Начать» или «Создать заказ через AI».
3. Перейти на `/register`. Сегмент «Регистрация» подсвечен.
4. Выбрать роль «Заказчик».
5. Ввести телефон `+7 900 100 00 01`. Маска корректно подставляется.
6. Пароль ≥ 8 символов. Принять условия оферты.
7. Нажать «Создать аккаунт». Должен произойти редирект на `/verify-phone`.
8. Ввести код `123456`. По завершении — редирект на `/onboarding`.

**Проверить:** на странице верификации появляется маскированный номер; код валидируется; ошибка показывается тостом.

## 2. Онбординг заказчика

1. Этап «Имя/компания» — заполнить.
2. Этап «Аватар» — загрузить файл < 5 МБ; превью корректное; «Сменить» / «Удалить» работают.
3. Этап «Описание» — описать профиль.
4. Этап «Баланс» — пополнить через тестовый депозит. Должен обновиться баланс.
5. По завершении — `PUT /profile` с `onboarding_completed_at`. Редирект на `/dashboard`.

**Проверить:** `GET /onboarding/state` возвращает `onboarding_completed: true`; повторный заход на `/onboarding` редиректит на `/dashboard`.

## 3. Создание заказа через AI

1. На `/dashboard/orders/new` левая панель — AI-чат, правая — собирающееся ТЗ.
2. Отправить сообщение «Нужен лендинг на Next.js, бюджет 80 000 ₽, срок 14 дней».
3. AI стримит ответ; в правой панели заполняются поля.
4. Заполнить вручную: заголовок, описание, категория, навыки (Tag-инпут с Enter), бюджет, дата (Filka-календарь, ru-локаль).
5. Если баланс < `budget_max` — клик «Опубликовать» открывает `InsufficientFundsModal` → пополнение → авто-повтор публикации.
6. После успеха — редирект на `/dashboard/orders/{id}` с заказом в статусе `published`.

## 4. Получение и принятие отклика

(требуется второй пользователь — фрилансер)

1. Зарегистрироваться как фрилансер (роль "Исполнитель"), пройти онбординг.
2. На `/dashboard/orders` (Биржа) видеть опубликованный заказ заказчика.
3. Открыть заказ, нажать «Откликнуться»; модалка с письмом, ценой и сроком.
4. Отправить отклик — статус `pending`. Реалтайм событие `proposal.created` приходит заказчику.
5. У заказчика на `/dashboard/orders/{id}` — отклик в списке. AI-баннер «Лучшее совпадение» (если бэк отдаёт).
6. Клиент жмёт «Принять» — `PUT /orders/:id/proposals/:pid/status` со status=`accepted`. Заказ → `in_progress`, эскроу → `held`.
7. Если на балансе недостаточно — открывается `InsufficientFundsModal` (через `parseInsufficientFundsError`).
8. Реалтайм у обеих сторон обновляет статусы.

## 5. Чат и завершение

1. Заказчик и фрилансер заходят в `/dashboard/messages/{conversationId}` (создаётся автоматически после `accepted`).
2. Видны pillsы AI («ПЛАН РАБОТЫ»), карточка эскроу справа.
3. Отправить сообщение Enter; Shift+Enter — перенос строки.
4. Реалтайм: пузырь приходит сразу второй стороне, типинг работает (`chat.typing.set`).
5. Файл прикладывается через `multipart /media/photos`.
6. Фрилансер жмёт «Сдать работу» (`POST /orders/:id/complete-by-freelancer`).
7. Заказчик жмёт «Принять и выплатить» (`POST /payments/escrow/:orderId/release`).
8. Эскроу освобождён; у фрилансера баланс +amount × (1 - feeRate); транзакции `escrow_release` отображаются в `/dashboard/balance`.

## 6. Уведомления

1. На `/dashboard/notifications` — группировка по датам, фильтры (Все / Непрочитанные / Сообщения / Заказы / Платежи / Система).
2. Колокол в топбаре показывает счётчик. Реалтайм (`notification.unread_count.updated`) синхронизирует.
3. «Прочитать всё» (`POST /notifications/read-all`) — счётчик обнуляется.
4. Клик по уведомлению открывает соответствующий объект (заказ/чат/баланс).

## 7. Баланс и транзакции

1. На `/dashboard/balance` видны: Доступно, На удержании, Заработано/Потрачено.
2. История транзакций с иконками типов (`deposit`, `withdrawal`, `escrow_hold`, `escrow_release`).
3. «Пополнить» — `FilkaModal` с быстрыми суммами.
4. «Вывести» — модал с last4 карты + банк (`POST /withdrawals`).
5. Эскроу-блок: список активных эскроу, кнопки «Освободить» / «Возврат» (для заказчика, через `POST /payments/escrow/:orderId/release|refund`).

## 8. Профиль / настройки

1. `/dashboard/profile` — редактирование имени, описания, ставки (фрилансер), сохранение через `PUT /profile`.
2. `/dashboard/profile/{id}` — публичный профиль; для других пользователей видны портфолио, отзывы, услуги.
3. `/dashboard/settings`:
   - Уведомления — `PUT /settings/notifications`.
   - AI — `PUT /settings/ai`.
   - Приватность — `PUT /settings/privacy` (видимость профиля, онлайн-статус, личные сообщения).
   - Сессии — `GET /auth/sessions` + завершение (`DELETE /auth/sessions/:id`).
   - Текущая сессия revoked → автологаут (`session.revoked` событие).

## 9. Realtime подключение

1. На `/dashboard` — индикатор `RealtimeStatusBar` появляется при потере соединения (отключить интернет).
2. После восстановления — индикатор пропадает.
3. WS принимает токен через subprotocol `ailance-ws, <token>`.

## 10. Адаптив

| Экран | Брейкпоинт | Что должно работать |
|-------|-----------|--------------------|
| 1440px | desktop | Топбар + горизонтальный dock + контент 1320px max |
| 1024px | laptop  | Топбар, dock компактный, контент сжимается |
| 768px  | tablet  | Скрывается dock, остаётся топбар + поиск |
| 390px  | mobile  | Топбар без dock, нижний `MobileBottomNav` (5 секций) |

## 11. Тёмная/светлая темы

- В `<html>` атрибут `data-theme="dark"` по умолчанию.
- Переключение через `data-theme="light"` (для будущей кнопки в настройках). Все цвета берутся из CSS-токенов.

## 12. Антианглицизмы

```bash
npm run lint:ru
```

Ожидаемый вывод: 0 строк или только подозрения внутри `${...}` шаблонных литералов и технических терминов.

## 13. TypeScript / тесты

```bash
npm run typecheck
npm run test
```

Должны проходить без ошибок.
